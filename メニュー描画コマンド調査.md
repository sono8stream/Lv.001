# メニュー描画コマンド調査

対象は B ボタンで起動する `common 127 / X[移]メニュー起動` から辿れる描画系コモンイベント群です。  
主に `64 / 88 / 90 / 102` を見ています。

## 今回の結論

- メニュー描画の検証に最低限必要だったのは、`99 / 124 / 150 / 290` のうち **メニュー系で頻出する variant の typed 化** でした。
- すべてを一度に実装するのではなく、まず **ウィンドウ/ピクチャ表示・移動・効果の土台** を入れました。
- これにより、メニュー実行時に **window-like な picture entry** が生成されるところまで確認できています。

## コマンド別メモ

### 99

- 名称: **チェックポイント**
- メニュー系では `[99, 0]` しか出現しておらず、実行意味は不要
- 対応: `checkpoint` として parse、runtime は no-op

### 124

- 名称: **変数操作＋**
- 今回のメニュー系出現分は **ピクチャ情報取得**
- 今回使った解釈:
  - `args[1]`: 代入先変数
  - `args[2]`: getter 設定 (`16384`, `17408`)
  - `args[3]`: pictureId
  - `args[4]`: 取得項目
- 今回対応した取得項目:
  - `0`: X
  - `1`: Y
  - `2`: 幅
  - `3`: 高さ
  - `5`: 不透明度

### 150

- 名称: **ピクチャ**
- メニュー系で重要だった variant:
  - **表示** (`low nibble = 0`)
  - **移動** (`low nibble = 1`)
  - **消去** (`low nibble = 2`, 一部 `3` も remove 相当で吸収)
- 今回対応したもの:
  - `showPictureString`: 文字列変数からファイルパスを引くピクチャ表示
  - `showWindowPicture`: 文字列トークンとサイズを使う簡易ウィンドウ表示
  - `movePicture`: 既存 picture entry の位置/倍率更新
- 既存対応と合わせ、メニューで使う表示・更新系のかなりの部分を吸収

### 290

- 名称: **ピクチャエフェクト**
- 今回のメニュー系では `32` が最多で、描画座標シフトとして使われていた
- 対応:
  - `pictureEffect` として typed 化
  - `effectType=32` のみ、現在位置への座標シフトとして runtime 実装
  - それ以外の effect type は現状 no-op

## 実装結果

- `checkpoint`
- `readPictureProperty`
- `showPictureString`
- `showWindowPicture`
- `movePicture`
- `pictureEffect`

以上を追加し、棚卸し上でも supported kind に反映済み。

## 今後の残件

残っている主要 key は次のとおり。

1. `106` アイテム/装備系データ処理
2. `112`, `140` SE 系
3. `221` セーブ/ロード補助
4. `124` の未対応 variant

描画系の次は、メニュー各欄の中身を進めるなら `106`、演出寄せを進めるなら `112/140` が自然。
