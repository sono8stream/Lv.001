# コマンド追加プロセス

今後の WOLF コマンド追加は、毎回この手順で進める。

## 手順

1. **対象コマンドを絞る**  
   - `コマンド棚卸し.md` と `Assets/StreamingAssets/command-inventory.json` を見て、件数と出現箇所を確認する。  
   - 今回のように特定機能（例: B メニュー）なら、起点コモンイベントから呼び出し閉包を辿って、その機能に本当に必要な key だけを対象にする。
2. **仕様リンクで裏を取る**  
   - `WOLF仕様リンク集.md` にある公式/非公式リンクから、対応表と引数構造を確認する。  
   - 少なくとも「コマンド名」「コード」「引数数」「制御フロー上の意味」を押さえる。
3. **テストを先に書く**  
   - parser テスト: `unknown` ではなく専用 kind へ変換されること  
   - runtime テスト: 実際の制御が期待どおり動くこと  
   - 可能なら実データ回帰: 実際のコモンイベント/マップイベントで落ちないこと
4. **typed command と runtime を実装する**  
   - `web/src/wolf/types.ts` に kind を追加  
   - `web/src/wolf/data.ts` で parser を追加  
   - `web/src/wolf/runtime.ts` で実行処理を追加
5. **棚卸しを更新する**  
   - `Assets/StreamingAssets/command-inventory.json` を再生成  
   - `コマンド棚卸し.md` を更新  
   - 必要なら機能別調査メモも更新する
6. **検証してコミット・push する**  
   - `cd web && npm run build`  
   - `cd web && npm run test:debug`  
   - タスク単位で commit / push

## 実装優先度の考え方

- **最優先**: 無限ループ回避、入力待ち継続、分岐継続などの制御系
- **次点**: データ参照/更新系
- **後回し可**: 効果音、演出、見た目のみの描画

## 今回の適用例

- 対象: `common 127 / X[移]メニュー起動`
- 抽出した最優先: `176 / ループ開始へ戻る`
- 実装結果: `loopContinue` として typed 化し、menu loop の continue 相当を runtime に追加
