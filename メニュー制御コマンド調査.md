# メニュー制御コマンド調査

対象は B ボタンで起動する `common 127 / X[移]メニュー起動` と、その先で名前呼び出しされるメニュー系コモンイベント群です。

## 今回の結論

- **メニュー制御の直接ブロッカーは `176` だった。**
- `176` はメニュー入力ループ中で使われており、これを素通りすると「カーソル移動後に入力待ちの先頭へ戻る」制御が崩れる。
- SE 系 (`112`, `140`) は後回しにし、描画系 (`99`, `124`, `150`, `290`) は別タスクで段階対応した。
- データ処理系 (`106`, `107`, `211`, `220`, `221`, `222`) はメニュー各機能の中身に関わるが、**メニュー制御そのものの最初の詰まり所** ではなかった。

## 優先度付き列挙

1. **176 / ループ開始へ戻る**  
   - 用途: メニュー入力ループの先頭へ戻る。上下キーでカーソルを動かした後や、選択不能項目で再入力待ちに戻る箇所で使われる。  
   - 出現例: `X[移]メニュー起動` の index `80`, `88`, `103`。  
   - 対応: **今回実装済み**。
2. **220 / 221 / 222 系**  
   - 用途: セーブ欄実行まわりで現れる。システムメニュー奥のセーブ・ロード導線で必要。  
   - 優先度: トップメニューの入力ループより後。
3. **106 / 107 / 211 系**  
   - 用途: アイテム・装備・使用効果など、各メニュー機能のデータ処理側。  
   - 優先度: 項目へ入った後の機能実装で必要。

## 仕様メモ

仕様リンク: `WOLF仕様リンク集.md` にある以下を参照。

- 非公式仕様: `イベントコマンド構造`  
  <https://kameske027.cloudfree.jp/woditor_analysis/pages/eventCommand.html>
- 非公式対応表: `イベントコマンド対応表`  
  <https://kameske027.cloudfree.jp/woditor_analysis/pages/eventCodeCorrespondence.html>

`176` について、`イベントコマンド構造` では次のように記載されている。

- 名称: **ループ開始へ戻る**
- コード: **`0xB0 00 00 00`**
- 数値引数: **0 個**
- 文字列引数: **0 個**
- 追加で持つのは **インデント深さ** のみ

このため Web 実装では、`176` を **現在の最内ループの先頭へ戻る control-flow command** として扱うのが妥当。

## 実装内容

- parser: `176` を `unknown` ではなく `loopContinue` に変換
- runtime: `loopContinue` を **現在のループの continue 相当** として実行
  - 無限ループならそのまま先頭へ戻る
  - 有限ループなら `loopEnd` と同じ回数進行で先頭へ戻る
  - 最終回で `loopContinue` が来た場合はループを抜ける

## 今回見送ったもの

- `112`, `140`: SE 系
- `99`, `124`, `150`, `290`: 描画/UI 演出系 → `メニュー描画コマンド調査.md` で別途対応

制御の成立を優先して先に `176` を入れ、その後に描画系を足す進め方を取った。
